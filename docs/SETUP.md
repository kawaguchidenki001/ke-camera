# セットアップ手順 ・ KE-Camera

KE-Camera を運用するために必要な準備手順をまとめます。完了までに **概ね 30 〜 45 分** を見込んでください。

## ゴール

設定が完了すると、以下の 4 つの値が手元に揃います。これを KE-Camera の「設定」画面に入力するだけで使えるようになります。

| 値 | 例 | 取得元 |
|---|---|---|
| OAuth クライアント ID | `xxxx-yyyy.apps.googleusercontent.com` | Google Cloud Console |
| Sheets API キー | `AIzaSy...` | Google Cloud Console |
| 工事マスタ シート ID | `1AbC...xyz` | Google Sheets の URL |
| 既定 Drive 保存先フォルダ ID | `1Foo...Bar` | Google Drive の URL |

---

## ステップ 1 ・ GitHub Pages へデプロイ(先に URL を確定する)

OAuth クライアント ID と API キーには「使える URL」を設定する必要があるため、**URL を先に決めます**。

1. GitHub で新規リポジトリを作成(例: `ke-camera`)。Public でも Private でも可
2. このプロジェクト一式を push
3. リポジトリの **Settings → Pages**
4. **Source: Deploy from a branch** を選択
5. **Branch: `main` / Folder: `/ (root)`** で保存
6. 数分待つと公開 URL が表示される(例: `https://kawaguchi-denki.github.io/ke-camera/`)

この URL を控えておきます。以降「**アプリ URL**」と呼びます。

> **メモ**: 末尾のスラッシュは含めても含めなくても OK ですが、後の HTTP リファラ制限では `/*` を付けます。

---

## ステップ 2 ・ Google Cloud プロジェクトの作成

1. <https://console.cloud.google.com/> にアクセス(初回は利用規約に同意)
2. 画面上部のプロジェクト選択 → **新しいプロジェクト**
3. プロジェクト名(例: `KE-Camera`)・組織はそのまま → **作成**
4. 作成完了したら、画面上部でそのプロジェクトに切り替え

---

## ステップ 3 ・ API の有効化

このアプリでは 2 つの API を使います。

1. ナビゲーション → **API とサービス → ライブラリ**
2. 検索ボックスで `Google Drive API` → ヒットしたものを開き **有効にする**
3. 同様に `Google Sheets API` → **有効にする**

---

## ステップ 4 ・ OAuth 同意画面の構成

1. **API とサービス → OAuth 同意画面**
2. **User Type** を選択
   - **外部** ← 取引先・協力会社も使う場合はこちら(本案件はこれ)
3. **作成** をクリック
4. 「アプリ情報」を入力
   - アプリ名: `KE-Camera 小黒板カメラ`(後でユーザーに表示される)
   - ユーザーサポートメール: 自分のメール
   - アプリのロゴ: 任意(`icons/icon-512.png` をアップロードしてもよい)
   - アプリのホームページ: アプリ URL
   - アプリのプライバシーポリシー URL: 任意(社外ユーザーが多い場合は用意推奨)
   - アプリの利用規約: 任意
   - 承認済みドメイン: `github.io`(Pages を使う場合)
   - デベロッパーの連絡先: 自分のメール
5. **次へ**
6. **スコープ** ページ
   - **スコープを追加または削除** をクリック
   - `https://www.googleapis.com/auth/drive.file` を **必ず**チェック(検索で `drive.file` と入力)
   - **更新**
   - 「非機密のスコープ」欄に `drive.file` が入っていれば OK。「機密のスコープ」「制限付きスコープ」欄が空であることを確認(空でないと検証が必要になる)
7. **次へ**
8. **テストユーザー** ページ → 一旦自分のメールアドレスを追加(後で本番公開する)
9. **次へ → 戻る**
10. 同意画面のステータスが **テスト中** になっている

### 本番公開(社外ユーザーに配布する直前に実施)

- OAuth 同意画面のトップで **アプリを公開** をクリック
- 確認ダイアログで「**確認**」
- `drive.file` だけを使っているので「**確認は不要です**」と表示されるはず。表示されたら完了

> **重要**: テスト中だと「テストユーザー」に登録した人しか使えません。本番公開すると誰でも使えますが、検証 (verification) は `drive.file` のみのため不要です。

---

## ステップ 5 ・ OAuth クライアント ID の作成

1. **API とサービス → 認証情報**
2. 上部の **+ 認証情報を作成 → OAuth クライアント ID**
3. **アプリケーションの種類: ウェブ アプリケーション**
4. 名前: `KE-Camera Web`(任意)
5. **承認済みの JavaScript 生成元** に追加:
   - `https://kawaguchi-denki.github.io`(`https://[ユーザー名].github.io`)
   - ローカルでテストする場合: `http://localhost:8080` も追加可
   > 末尾にスラッシュは付けない。**オリジン**を入力
6. **承認済みのリダイレクト URI** は **空のままで OK**(GIS の token client はリダイレクト不要)
7. **作成**
8. ダイアログに表示された **クライアント ID** を控える(`xxxx.apps.googleusercontent.com`)

---

## ステップ 6 ・ API キーの作成(Sheets 用)

1. **API とサービス → 認証情報**
2. **+ 認証情報を作成 → API キー**
3. 作成された API キーをコピーして控える
4. すぐに **キーを制限** を開く
5. **アプリケーションの制限**: **HTTP リファラー(ウェブサイト)**
   - **ウェブサイトの制限** に追加:
     - `https://kawaguchi-denki.github.io/*`
     - 必要に応じて `http://localhost:*/*`(開発用)
6. **API の制限**: **キーを制限**
   - **Google Sheets API** のみチェック
7. **保存**

> **重要**: HTTP リファラ制限を忘れると、第三者にキーが流出した場合に API クォータを悪用される可能性があります。**必ず**設定してください。

---

## ステップ 7 ・ 工事マスタ Google Sheets の作成

1. <https://sheets.google.com/> で **新規スプレッドシート** を作成
2. ファイル名を **「KE-Camera 工事マスタ」**(任意)に変更
3. シート名(タブ)を **「工事一覧」** に変更
4. 1 行目にヘッダーを入力(下記の通り)
5. 2 行目以降に工事を追加(例も以下に記載)

### 列構造

| 列 | 内容 | 必須 | 備考 |
|---|---|---|---|
| A | 工事名 | ✅ | アプリで表示する正式名 |
| B | 工種 | | 既定値(編集可) |
| C | 施工者 | | 既定値(編集可) |
| D | Drive フォルダ ID | | この工事専用の保存先(未指定なら設定の既定フォルダ) |
| E | 区分 | | 任意。ラベル表示用(例: 公共・私企業など) |

### サンプル

```
A                                                B                  C            D                                                  E
工事名                                            工種                施工者        Drive フォルダ ID                                  区分
県営北方住宅室内照明LED化改修工事                LED 化改修          河口電機株式会社  1AbcDefGhi...                                       公共
北保健センター電気設備改修工事                   電気設備工事        河口電機株式会社  1BcdEfgHij...                                       公共
○○ビル LED 化更新工事                          照明器具取替        河口電機株式会社                                                    民間
```

> Drive フォルダ ID は **「フォルダ毎の URL の末尾文字列」**。後の手順で取得します。

### 共有設定 ⚠️ 重要

このシートは「API キー読み取り」で参照するため、**「リンクを知っている全員」が閲覧できる**必要があります。

1. 右上の **共有** をクリック
2. 「一般的なアクセス」を **リンクを知っている全員** に変更
3. 権限は **閲覧者** のままでよい
4. **完了**

> セキュリティ上は「URL を知らない人には到達できない」設計です。シートに極めて機密な情報を入れない運用にしてください。

### シート ID の取得

URL がこのような形式になっています:

```
https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz/edit#gid=0
                                       ─────────────┬─────────────
                                            ↓ これがシート ID
```

`/d/` と `/edit` の間の文字列を控えます。KE-Camera の設定画面では URL 全文を貼っても自動抽出してくれます。

## ステップ 8 ・ Google Drive 保存先フォルダの準備

工事ごと、または共通の保存先フォルダを Drive に作成します。

### A) 共通フォルダで運用(シンプル)

1. Drive で新規フォルダ作成(例: `KE-Camera 工事写真`)
2. URL `https://drive.google.com/drive/folders/1XyZAbC...` の `/folders/` 以降を控える ← これがフォルダ ID
3. 設定画面の **既定保存先フォルダ ID** に貼り付け

### B) 工事ごとに分けて運用(推奨)

1. 各工事のフォルダ(例: `[1] 県営北方住宅 LED 化`)を作成
2. 各フォルダ ID を取得
3. 工事マスタ Sheets の **D 列** に貼り付け

### 取引先と共有する場合

- そのフォルダの **共有** で取引先の Google アカウントを「**編集者**」として追加
- KE-Camera は `drive.file` スコープのため、**「アプリが作った/開いたファイル」だけ**にアクセス可能。フォルダ全体には触れない(アップロードだけ可能)

> `drive.file` の制約により、アプリは「親フォルダ ID を指定してファイル作成」はできるが、フォルダ内のファイル一覧は取得できない(他のアプリで作った写真が見えない)。これは仕様です。

---

## ステップ 9 ・ KE-Camera の起動と設定入力

1. アプリ URL を開く
2. 右下の「**設定を開く**」 or 左上のメニュー → 設定
3. 以下を入力 → **保存**
   - OAuth クライアント ID: ステップ 5 で取得
   - Sheets API キー: ステップ 6 で取得
   - スプレッドシート ID: ステップ 7 で取得(URL 全文を貼っても OK)
   - 読み取り範囲: `工事一覧!A2:E`(既定値のまま)
   - 既定保存先フォルダ ID: ステップ 8 で取得
   - ファイル名テンプレート: 既定値か、運用に合わせて変更
   - 既定の施工者名: 河口電機株式会社 など
4. ホームに戻り **Google でログイン**
5. ポップアップで Google アカウントを選び、表示されるスコープ(`drive.file`)を確認して許可
6. ヘッダー右上のドットが緑になれば認証完了

## トラブルシュート

### 「Sheets API エラー (403)」と出る
- シートが「リンクを知っている全員」公開になっていない
- API キーの HTTP リファラ制限が間違っている
- API キーで Sheets API が許可されていない

### 「カメラの使用が許可されていません」
- ブラウザのアドレスバー左のロックアイコンからカメラ許可を確認
- iOS の場合、設定 → Safari → カメラ も確認

### ログインで「このアプリは確認されていません」と出る
- OAuth 同意画面のステータスが「テスト中」のまま
- ステップ 4 末尾の「**アプリを公開**」を実施
- それでも出る場合は、テストユーザー(自分のメール)に追加してログイン

### Drive 保存で 404
- フォルダ ID の誤り、または `drive.file` スコープでアクセスできないフォルダ
- 別のアプリで作ったフォルダで権限が無い場合 → そのアプリ経由で当アプリを「追加」する必要があるケースあり
- いったん **自分の Drive 直下に作った新規フォルダ**で動作確認

### Drive 保存で 401
- アクセストークン期限切れ。トースト指示に従って再ログイン

---

## 運用上の注意

- **API キー は公開リポジトリにコミットしない**。KE-Camera は localStorage にユーザーごとに保存する設計なので、コードに埋め込む必要はありません
- **クライアント ID は公開しても問題ない**(JavaScript 生成元の制限で守られる)
- **マスタシートに過剰な機密情報を入れない**。共有範囲を意識した運用を
- **本番公開後にスコープを変えると検証が必要**になる場合があるので、最初に必要なスコープを決めること
- 写真は工事フォルダに溜まり続けるので、定期的に整理する運用も検討
