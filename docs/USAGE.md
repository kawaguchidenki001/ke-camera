# 使い方 ・ KE-Camera

セットアップ完了後の、日常的な使い方を説明します。

## 起動

- スマホ・タブレットならホーム画面のアイコンから起動
- PC ならブラウザでアプリ URL を開く
- 起動時に右上のドットが緑なら認証済み・赤なら未認証

## 基本フロー(連続撮影)

```
①ログイン → ②工事を選ぶ → ③黒板を入力 → ④撮影 → ⑤Drive に保存
                                          ↑              │
                                          │←─ 連続撮影 ──┘
```

撮影後 1 タップで Drive に保存され、自動的にカメラ画面に戻ります。同じ工事を連続で撮るときに最も速い動線になっています。

---

## 各画面の使い方

### ホーム

- **Google でログイン** … 初回は同意画面が出るので `drive.file` 権限を許可
- **設定を開く** … 設定値の編集
- 未設定項目があればチェックリストで表示されます

### 工事を選択

- 設定したマスタシート(Google Sheets)から工事一覧を取得して表示
- 検索ボックスで工事名・工種・施工者でフィルタ可能
- 右上の更新ボタン(↻)で再読み込み
- マスタを使わず手入力で進めたい場合は下部の「マスタを使わず手入力で進める」

### 黒板の内容

撮影前にこの内容を確認します。

| 項目 | 説明 |
|---|---|
| 工事名 | マスタ選択時は自動入力(編集可) |
| 工種 | 候補リストあり。自由入力可 |
| 施工者 | マスタ or 設定の既定値を初期表示 |
| 撮影内容 | この写真の説明(例: 1号棟廊下 灯具設置状況) |
| 撮影年月日 | 今日の日付が初期値 |

**表示オプション(必要に応じて変更)**

- 黒板の位置: 写真の下 / 上
- 黒板の高さ比率: 小(22%)/ 標準(28%)/ 大(34%)
- 連番表示: ON / OFF(連番は撮影ごとに自動採番)

**「撮影画面へ進む」** で次へ。

### カメラ

- 上に現在の **工事名 ・ 解像度 ・ 次の連番** を表示
- 中央の **白い丸ボタン** がシャッター
- 左下: 戻る / 右下: 前面 ⇄ 背面カメラ切替
- シャッターを押すと黒板を焼き込んだ画像が生成され、プレビューへ

### プレビュー

- 焼き込み後の画像を全面表示
- 上部にファイル名と保存先状態を表示
- **撮り直し** で破棄してカメラへ戻る
- **Drive に保存** でアップロード開始

### 設定

セットアップ手順の値を入力・編集する画面。

- 認証関連: OAuth クライアント ID
- 工事マスタ: API キー / シート ID / 読み取り範囲
- Drive 保存先: 既定フォルダ ID / ファイル名テンプレート
- 撮影者(既定値): 既定の施工者名
- データ: 設定の書き出し / 読み込み / 初期化

## ファイル名テンプレート

設定画面の「ファイル名のテンプレート」で使えるトークン:

| トークン | 展開内容 | 例 |
|---|---|---|
| `{date}` | 撮影年月日(8 桁) | `20260624` |
| `{time}` | 撮影時刻(6 桁) | `094215` |
| `{kouji}` | 工事名 | `県営北方住宅...` |
| `{koushu}` | 工種 | `LED化改修` |
| `{naiyou}` | 撮影内容 | `1号棟1階廊下` |
| `{seq}` | 連番(3 桁) | `005` |

既定値: `{date}_{kouji}_{naiyou}_{seq}.jpg`

例 → `20260624_県営北方住宅室内照明LED化改修工事_1号棟1階廊下_005.jpg`

> ファイル名に使えない文字(`\ / : * ? " < > |`)は自動的に除去されます。

## 連番について

- 工事 × 日付 ごとに 1 から自動採番
- 端末の localStorage に保存(端末をまたいだ連番共有はされない)
- 日付が変わると 1 にリセット
- アップロード失敗時は採番を巻き戻す(欠番を防ぐ)
- 「設定 → データ → 設定を初期化」では連番もリセットされる

複数端末で同じ工事を撮る場合は、端末ごとに採番が独立するので、ファイル名が衝突する可能性があります。その場合はファイル名テンプレートに `{time}` を入れる運用を推奨。

## 工事マスタの編集

工事を追加するときは Google Sheets を直接編集してください。

1. KE-Camera 工事マスタ シートを開く
2. 最終行の下に新しい行を追加
3. A: 工事名 / B: 工種 / C: 施工者 / D: Drive フォルダ ID / E: 区分
4. KE-Camera の工事選択画面で **↻** ボタンを押すと反映

---

## 撮り溜めた写真の確認

撮影した写真は Drive 上の指定フォルダに溜まります。

- ブラウザ: <https://drive.google.com/>
- Drive アプリ(iOS / Android)
- 工事マスタ Sheets の D 列にフォルダ ID が入っていれば、その URL を直接開ける(例: `https://drive.google.com/drive/folders/<ID>`)

写真のメタデータ(description / properties)には、撮影時の黒板情報がすべて含まれているので、Drive 検索で後から特定の工事名・撮影内容を探すこともできます。

---

## オフライン時の挙動

- アプリ本体(HTML / JS / CSS / アイコン)は Service Worker でキャッシュ済みなので、オフラインでも起動できる
- カメラ撮影と黒板の焼き込みもオフラインで完結
- ただし **Drive へのアップロードはオンライン時のみ**
- 工事マスタの読み込みも初回はオンライン必須(2 回目以降は前回取得分を保持)

> 現バージョンは「オフライン時に撮った写真を後でまとめてアップロード」の機能は未実装。電波がない現場では、保存して持ち帰り、車内などでアップロードする運用になります。

## トラブル対応

### 黒板の文字が画像からはみ出る・小さすぎる
- 黒板高さ比率を変える(小 / 標準 / 大)
- 工事名・撮影内容が極端に長い場合は末尾が「…」で省略されます

### 同じ工事で 2 回撮ったのに連番が飛んだ
- 別の端末で撮った可能性。連番は端末ローカルです
- またはアップロード失敗で巻き戻された可能性

### 設定をなくしてしまった
- 設定 → データ → 「設定を書き出し」で JSON ファイルをバックアップしておくのが安全
- 端末の OS 再インストール・キャッシュ消去で localStorage は消えます

### 別の端末・別のユーザーに設定を渡したい
- 設定の書き出し JSON を共有 → 受け取り側で「設定を読み込み」
- 注意: API キー も含まれるので、信頼できる相手・経路のみで共有

### 認証エラーが繰り返し出る
- 設定 → メニュー → サインアウト → 再ログイン
- それでも駄目なら設定 → 認証情報を再確認
